Introduction

Library Checker 是一个模板题库，包含了数百道模板题，例如多项式乘法、区间最小值等，它的数据生成器等全部公开在 GitHub 上，这意味着用户可以在本地验证代码的正确性，做基准测试、覆盖测试；也可以做其他有趣的事情，比如比较哪个操作系统更快、哪个编译器更强、哪些编译选项优化效果更好等等。如果想添加新的模板题，或是发现数据有待加强，题库也欢迎用户做贡献。

关于如何做基准测试，可以参考 Benchmarking tips.

本仓库包含了我的题解。本文档包含了所有题解的基准测试和覆盖测试结果。

基准测试使用time和perf命令测量程序的性能。unionfind 提供了一个例子。由于题解能够处理多组数据，所以提供了串行处理所有数据的测试点ALL，预期的情况是ALL的时间占用略小于Sum，空间占用略大于Max。基准测试对底层优化非常有帮助。

覆盖测试分为两种，一种是 LCOV 生成的全局覆盖测试 (link)，预期的情况是所有题解的覆盖率都接近 100%；另一种是针对单个测试点的覆盖测试，例如在树链剖分问题中，注意到大多数时候都处理一条树链的前缀，所以可以在这个基础上优化线段树，然后进行覆盖测试。对于最慢的那个测试点，对比 main 和 slow，可以发现对于线段合并操作，前者的次数是后者的 64%。覆盖测试对算法优化非常有帮助。
